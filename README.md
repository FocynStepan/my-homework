# Контрольні запитання

### 1. Що таке дизайн у широкому сенсі?
Це не те, як предмет виглядає, а те, як він працює. У широкому сенсі це процес вирішення проблеми. Це планування та створення чогось (чи ручка, чи інтерфейс сайту) так, щоб воно виконувало свою функцію максимально ефективно, зручно та естетично для користувача.

### 2. Поясніть різницю між UI та UX на прикладі ліфта.
* **UI (User Interface):** Це те, що я бачу. Панель із кнопками, їхній колір, підсвітка поверхів, дзеркало в кабіні, шрифт, яким написані цифри.
* **UX (User Experience):** Це те, що я відчуваю і як це працює. Чи швидко приїхав ліфт? Чи зрозуміло, яку кнопку тиснути, щоб двері не закрилися (бо часто іконки >|< та <|> плутають)? Чи не занадто різко він гальмує? Якщо я натиснув кнопку, а вона не засвітилася, і я не розумію, чи прийняв ліфт команду — це поганий UX.

### 3. Які обмеження накладає сенсорний екран смартфона порівняно з монітором ПК?
Головний біль мобільної розробки:
* **Відсутність стану Hover:** Ми не можемо навести курсор, щоб побачити підказку. Елемент або натиснутий, або ні.
* **«Fat fingers»:** Пальцем важче влучити в дрібні елементи, ніж курсором миші. Тому зона натискання (tap target) має бути мінімум 44x44 пікселі.
* **Робочий простір:** Екран малий, контенту влазить менше, треба ховати меню в «гамбургери».
* **Зовнішні умови:** Сонце може відсвічувати (контрастність важлива), а користувач може тримати телефон однією рукою на ходу.

### 4. Що таке «афорданс» в інтерфейсі?
Це підказка про використання об'єкта, закладена в його формі. Випукла кнопка з тінню «каже»: «Натисни мене». Синій підкреслений текст — це посилання. Сіре поле введення — неактивне.

### 5. Як можна класифікувати інтерфейси?
* CLI (Command Line Interface): Консоль, тільки текст.
* GUI (Graphical UI): Вікна, іконки, меню (як Windows).
* Сенсорні та NUI: Жести, мультитач, природні рухи.

# Питання на дослідження

### Завдання 1. 10 принципів хорошого дизайну за Дітером Рамсом
Дітер Рамс — це легенда (дизайнер Braun), на якого рівняється Apple. Ось його заповіді з моїми коментарями:
1. **Інноваційний.** Не копіювати старе, а використовувати нові технології для покращення.
2. **Корисний.** Продукт купують, щоб ним користуватися. Все, що заважає зайве.
3. **Естетичний.** Речі, якими ми користуємося щодня впливають на наш настрій тому потрібна краса щоб радувати око.
4. **Зрозумілий.** Дизайн має сам пояснювати, як користуватися продуктом (інтуїтивність керування).
5. **Ненав'язливий.** Продукти, які служать на користь якоїсь мети схожі на інструменти. Інструменти не повинні бути як витвори мистецтва. Вони мають бути нейтральними, щоб ними просто можна було користуватися(це в подальшому приводить до звички якщо це програма/сайт).
6. **Чесний.** Не обіцяти функцій, яких немає. Не маніпулювати користувачем.
7. **Довговічний.** Мода минає, хороший дизайн залишається актуальним десятиліттями.
8. **Продуманий до дрібниць.** Нічого випадкового. Повага до користувача — це увага до деталей.
9. **Екологічний.** Економія ресурсів, мінімум візуального та фізичного забруднення.
10. **Якнайменше дизайну.** Концентрація на суті, а не на декорі.

### Завдання 2. Инклюзивність в контексті ситуації в Україні
Зараз ця тема в Україні перейшла з розряду «було б непогано впровадити» в розряд «критично необхідно».
* **Порушення моторики:** Багато військових та цивільних втратили кінцівки. Інтерфейси мають бути адаптовані під керування однією рукою (важливі кнопки внизу екрана) або голосове керування(якщо це можливо).
* **Порушення зору:** Через поранення зростає кількість людей з вадами зору. Сайти держпослуг (Дія, Helsi) мусять ідеально працювати зі скрінрідерами (VoiceOver, TalkBack).
* **Умови блекаутів:** Це специфічний український контекст. Дизайн має бути легким (швидко вантажитися при поганому зв'язку E/2G) і мати Dark Mode (економія заряду батареї).
* **Психологічний стан:** Люди в стресі. Інтерфейси оповіщень про тривогу не повинні викликати паніку "червоним миготінням", але мають бути чіткими та однозначними.

### Завдання 3. Приклади невдалих інтерфейсних рішень
1. **Skype/Zoom (Кнопка виходу):** Часто кнопки "Mute" і "Leave Call" знаходяться поруч або змінюють положення. Ти хочеш вимкнути звук, а випадково кидаєш трубку під час важливої пари.
2. **Instagram (Зміна розташування кнопок):** Кілька разів Instagram міняв місцями кнопку створення нового посту (+) та кнопку Reels або Shop у нижньому меню. Користувачі звикли роками тиснути в одне місце, щоб додати фото, а тепер потрапляють у магазин. Це порушення звички та поганий UX.
3. **Форми реєстрації з жорсткими вимогами до пароля без підказок:** Коли ти вводиш пароль, а система каже просто "Помилка". І ти маєш вгадати: чи це немає великої літери, чи спецсимволу, чи він закороткий. Хороший дизайн одразу показує список вимог і ставить галочки, коли ти їх виконуєш у реальному часі.

### Завдання 4. 6 видів UI-інтерфейсів

**1. GUI (Graphical User Interface) — Графічний інтерфейс.**
* **Опис:** Найпоширеніший тип. Використовує візуальні метафори: вікна, іконки, меню, кнопки та курсор. Взаємодія відбувається через маніпуляції з графічними об'єктами.
* **Приклад в ІТ:** Операційні системи Windows, macOS, будь-яка десктопна програма (наприклад, Photoshop).

**2. CUI (Character User Interface / Composite UI (псевдографіка в консолі) / Conversational UI (чат-боти)).**
* **Опис:** Розгляну як Conversational UI. Це інтерфейс у формі діалогу, чату.
* **Приклад в ІТ:** Чат-боти в Telegram для техпідтримки, ChatGPT, інтерфейс банківських застосунків у вигляді чату.

**3. CLI (Command Line Interface) — Інтерфейс командного рядка.**
* **Опис:** Користувач вводить текстові команди, комп'ютер відповідає текстом. Немає графіки, тільки текст.
* **Приклад в ІТ:** Термінал Linux/macOS, командний рядок Windows (cmd), Git Bash. Основний інструмент для системних адміністраторів, DevOps та розробників.

**4. NUI (Natural User Interface) — Природний інтерфейс.**
* **Опис:** Інтерфейс, який прагне бути "невидимим". Керування відбувається природними для людини діями: жестами рук, рухом тіла, поглядом.
* **Приклад в ІТ:** Ігрові сенсори Kinect, керування жестами в VR-шоломах (Oculus), Apple Vision Pro (керування поглядом і щипком пальців).

**5. TUI (Tangible User Interface / Touch UI) — Тактильний/Сенсорний інтерфейс.**
* **Опис:** Взаємодія через дотики до екрану. Включає тапи, свайпи, пінчі (щипки для зуму).
* **Приклад в ІТ:** Смартфони (iOS, Android), планшети, сенсорні кіоски самообслуговування (McDonald's, термінали оплати).

**6. VUI (Voice User Interface) — Голосовий інтерфейс.**
* **Опис:** Керування системою за допомогою голосових команд. Потребує розпізнавання мови. Дозволяє взаємодію "без рук".
* **Приклад в ІТ:** Голосові асистенти Siri, Google Assistant, Alexa, голосовий пошук в YouTube, розумні колонки.
